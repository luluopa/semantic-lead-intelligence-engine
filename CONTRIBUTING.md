# Contributing to Lead Enrichment System

Thank you for contributing! To maintain the quality of the project, please follow these guidelines.

## Development Workflow
1. Follow the architecture defined in [AGENTS.md](./AGENTS.md).
2. Ensure all code is linted and formatted (`npm run lint`, `npm run format`).
3. Write tests for every new feature.
4. Update the documentation if you change any API behavior.

## Commit Message Convention
We follow a simple conventional commits style:
- `feat:` for new features.
- `fix:` for bug fixes.
- `docs:` for documentation changes.
- `test:` for adding or updating tests.
- `chore:` for maintenance tasks.

## AI Assistance
If you are using an AI agent (Cursor, GitHub Copilot, etc.), ensure it has read the `AGENTS.md` file to align with our architectural decisions.
